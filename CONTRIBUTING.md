# Contributing

1. Create a focused branch from `main`.
2. Never commit `.env` files, credentials, production customer data, or generated build output.
3. Run `npm run verify` from the repository root.
4. Add or update tests for behavior changes.
5. Open a pull request describing the user impact, technical approach, verification performed, and deployment considerations.

Commits should be small and use clear imperative or Conventional Commit-style messages. Pull requests must pass CI and CodeQL before merging.
